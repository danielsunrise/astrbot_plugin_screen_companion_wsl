# 我会一直看着你 (astrbot_plugin_screen_companion)

## 插件简介
`astrbot_plugin_screen_companion` 是一款智能屏幕观察插件，通过截图分析和环境感知，为用户提供个性化的互动体验。它不仅能观察用户的屏幕活动，还能感知环境变化，如系统状态、天气情况和节假日，甚至能监听麦克风音量，为用户提供全方位的智能陪伴。

**版本：1.5.0**
**GitHub 仓库：** [https://github.com/menglimi/astrbot_plugin_screen_companion](https://github.com/menglimi/astrbot_plugin_screen_companion)

---

## 功能说明
- **主动屏幕洞察**：根据配置的时间间隔，自动捕获当前电脑屏幕。
- **视觉理解与反馈**：调用视觉大模型分析截图内容，先识别屏幕内容，再基于识别结果进行互动。
- **个性化互动**：支持自定义人格（如：毒舌教练、温柔助理、逗比队友），对用户行为进行多维度的回应。
- **手动触发**：提供即时触发指令，想听吐槽时随时召唤。
- **灵活控制**：可随时开启或关闭自动监控模式，并调整截图频率。
- **随机窥屏**：支持设置随机因子，使自动窥屏更加随机。
- **时间段设置**：支持设定自动窥屏的活跃时间段，支持跨天设置。
- **互动模式**：提供四种预设互动模式（轻度、中度、高频、静默），满足不同场景需求。
- **环境感知**：
  - 节假日识别：在特殊节日提供相应的互动内容
  - 天气感知：根据天气情况调整对话内容
  - 系统状态检测：检测系统资源使用情况，提醒用户休息
- **日记功能**：开启后，每天在设定的时间生成风格化的日记，记录当天的观察内容。
- **日记参考功能**：生成日记时可以参考前几天的日记内容，保持风格的连贯性。
- **自定义监控任务**：在配置文件中设置特定时间的监控任务，如凌晨2点提醒休息。
- **麦克风监听**：监听麦克风音量，当音量超过阈值时触发截屏回复，及时响应用户的声音活动。
- **用户行为学习**：根据用户的反馈和行为模式调整互动方式，提供更加个性化的体验。

## 插件流程
该插件的工作逻辑分为**后台监控**和**消息处理**两个核心部分：

1.  **初始化阶段**：
    - 插件启动时，从配置加载用户设置（包括是否开启、截图间隔、系统提示词等）。
    - 启动后台任务在后台循环运行，包括自动监控任务、日记任务、自定义监控任务和麦克风监听任务。

2.  **自动触发逻辑**：
    - 后台任务检查 `enabled` 状态和当前时间是否在活跃时间段内。
    - 若开启且在活跃时间段内，则进入等待期（基于配置的间隔）。
    - 等待结束，调用 `pyautogui` 获取当前屏幕快照。

3.  **视觉识别与大模型调用**：
    - 插件调用外接视觉API识别屏幕内容。
    - 基于识别结果，结合人格设定进行针对性互动。
    - 如果是自动触发，插件会使用管理员QQ号或配置的主动推送目标，通过 `self.context.send_message` 主动推送识别结果。

4.  **手动指令流程**：
    - 用户发送 `/kp`。
    - 插件立即执行截图、分析、并直接返回回复。

5.  **日记功能**：
    - 开启后，插件会记录每天的观察内容。
    - 在设定的时间生成风格化的日记，并发送给指定目标。
    - 可配置参考前几天的日记内容，保持风格的连贯性。

6.  **自定义监控任务**：
    - 插件会定期检查配置的自定义监控任务。
    - 当到达设定的时间时，执行屏幕分析并发送提醒。

7.  **麦克风监听**：
    - 插件会定期检查麦克风音量。
    - 当音量超过阈值时，触发屏幕分析并发送回复。

8.  **用户行为学习**：
    - 插件会记录用户的反馈和行为模式。
    - 根据学习数据调整互动方式，提供更加个性化的体验。

9.  **清理阶段**：
    - 任务完成后，插件会自动清理临时的截图文件，确保不占用过多的磁盘空间。
    - 生成日记后，会清空当天的观察记录，为第二天做准备。

## 使用方法

### 指令说明
| 指令 | 参数 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `/kp` | 无 | 立即截取当前屏幕并让机器人进行点评。 | `/kp` |
| `/kps` | 无 | 切换自动监控的状态（开启/关闭）。 | `/kps` |
| `/kpi start` | 无 | 启动自动监控任务。 | `/kpi start` |
| `/kpi stop` | 无 | 停止所有自动监控任务。 | `/kpi stop` |
| `/kpi list` | 无 | 列出所有运行中的自动监控任务。 | `/kpi list` |

### 场景示例
- **学习/工作辅助**：设置人格为"严厉导师"，当你打开社交媒体摸鱼时，机器人会及时提醒你回到工作。
- **游戏互动**：当你打出精彩操作或屏幕黑掉时，手动触发 `/kp` 让机器人为你喝彩或无情嘲笑。
- **随机监督**：设置合理的随机因子和活跃时间段，让机器人在你工作时间内随机检查你的屏幕，增加工作效率。
- **环境感知**：在节假日收到特殊的问候，根据天气情况获得相应的建议。
- **日记记录**：开启日记功能，每天自动生成风格化的日记，记录你的日常活动。
- **定时提醒**：在配置文件中设置凌晨2点的监控任务，提醒你注意休息。
- **声音响应**：当你说话声音较大时，机器人会自动截屏并询问你发生了什么事。
- **个性化学习**：通过用户行为学习，机器人会逐渐了解你的喜好和习惯，提供更加个性化的互动内容。

## 配置说明
插件支持通过管理面板进行配置（基于 `_conf_schema.json`）：

### 基本设置
- `enabled`: (bool) 是否默认开启自动截图互动。
- `interaction_mode`: (string) 互动模式，选择预设的互动模式或自定义。
- `check_interval`: (int) 检查间隔，每隔多少秒进行一次触发判定，最小值为10秒。
- `trigger_probability`: (int) 触发概率，每次判定时触发的概率（0-100）。
- `active_time_range`: (string) 活跃时间段，格式为"HH:MM-HH:MM"，例如"09:00-22:00"。支持跨天设置，例如"22:00-06:00"。留空表示全天活跃。

### 人格与互动
- `system_prompt`: (text) 设定机器人的人格。例如："你是一个毒舌的屏幕评论员，请根据图片内容犀利地吐槽用户的行为。"
- `watch_mode`: (string) 观察模式，选择"观看"模式（正常观察）或"偷看"模式（偷偷瞄了一眼）。

### 截图与视觉
- `capture_mode`: (string) 截图区域，选择截取全屏还是活动窗口。
- `image_quality`: (int) 截图压缩质量，设置1-100之间的整数。较低的质量可以减少网络流量消耗和AI响应延迟。
- `image_prompt`: (text) 图像识别提示词，自定义LLM识别图像时的提示词，以获得更好的识别效果。

### 外接API
- `use_external_vision`: (bool) 是否使用外接视觉API，默认开启。
- `vision_api_url`: (string) 外接视觉API的URL地址，例如：https://api.qhaigc.net/v1/chat/completions。
- `vision_api_key`: (string) 外接视觉API的密钥。
- `vision_api_model`: (string) 外接视觉API使用的模型名称，例如：gpt-5-mini。

### 环境感知
- `weather_api_key`: (string) 天气API密钥，用于获取天气信息。留空则不启用天气感知功能。
- `weather_city`: (string) 城市名称，用于获取天气信息。

### 日记功能
- `enable_diary`: (bool) 启用日记功能，开启后，bot会在每天设定的时间生成风格化的日记。
- `diary_time`: (string) 日记生成时间，格式为"HH:MM"，例如"22:00"。
- `diary_storage`: (string) 日记存储路径，留空则使用默认路径。
- `diary_reference_days`: (int) 生成日记时参考前几天的日记内容，0表示不参考，1表示参考前1天，以此类推。

### 自定义监控任务
- `custom_tasks`: (text) 设置特定时间的监控任务，格式为每行一个任务：时间 提示词。例如：02:00 这么晚了还在工作，快去休息吧！

### 麦克风监听
- `enable_mic_monitor`: (bool) 启用麦克风监听，开启后，插件会监听麦克风音量，当音量超过阈值时触发截屏回复。
- `mic_threshold`: (int) 麦克风音量阈值，麦克风音量超过此值时触发截屏回复，范围 0-100。
- `mic_check_interval`: (int) 麦克风检查间隔，每隔多少秒检查一次麦克风音量，最小值为 1 秒。

### 用户偏好设置
- `user_preferences`: (text) 设置不同场景的互动风格，格式为每行一个场景：场景名称 互动风格描述。例如：编程 严厉的导师风格，催促用户专注编码。
- `enable_learning`: (bool) 启用用户行为学习，开启后，插件会根据用户的反馈和行为模式调整互动方式。
- `learning_storage`: (string) 学习数据存储路径，用户行为学习数据的存储路径，留空则使用默认路径。

### 消息与通知
- `admin_qq`: (string) 管理员QQ号，用于接收主动消息和通知。
- `proactive_target`: (string) 主动消息推送目标，格式为"aiocqhttp:private_message:123456"。如果未设置，插件将使用管理员QQ号构建目标。
- `save_local`: (bool) 是否保留最近一次的截图在本地。

## 注意事项
1.  **环境要求**：
    - 运行 AstrBot 的环境必须是**带有图形界面的桌面环境**（Windows/macOS/带桌面的Linux）。
    - 需确保已安装 `pyautogui` 和 `Pillow` 库（通常由插件自动处理）。
    - 启用麦克风监听功能时，需安装 `pyaudio` 和 `numpy` 库。
2.  **隐私提醒**：
    - 插件会捕获您的实时屏幕图像并发送至配置的 LLM 提供商 API。**请勿在处理涉及个人隐私、账号密码、财务信息等敏感信息时开启此功能。**
    - 截图内容可能包含您的个人信息、聊天记录、浏览历史等敏感数据，请确保您信任所使用的 LLM 提供商。
    - 建议在公共环境或他人使用您电脑时关闭此插件。
    - 麦克风监听功能会捕获环境声音，可能包含私人对话，请谨慎使用。
    - 请定期检查插件的配置和权限设置，确保仅在必要时启用相关功能。
3.  **API配置**：
    - 请确保您正确配置了外接视觉API的URL、密钥和模型名称。
    - 推荐使用支持多模态的模型，如 gpt-5-mini、GPT-4o、Claude 3.5 Sonnet、Gemini 1.5 Pro 等。
4.  **权限**：
    - 在 macOS 或 Windows 上，运行 AstrBot 的终端/IDE 可能需要"屏幕录制"权限，请在系统设置中予以授权。
    - 启用麦克风监听功能时，可能需要麦克风访问权限。
5.  **合规性**：
    - 严禁将此插件用于任何非法监控或侵犯他人隐私的用途。请在遵守当地法律法规的前提下使用。
6.  **安装与更新**：
    - 从 GitHub 仓库获取插件：`git clone https://github.com/menglimi/astrbot_plugin_screen_companion.git`
    - 将插件目录复制到 AstrBot 的插件目录中：`C:\Users\用户名\.astrbot\data\plugins\`
    - 重启 AstrBot 以加载插件
